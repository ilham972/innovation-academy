# Tuition Center Timetable Management — Full Build Specification

> **Purpose of this document:** This is a complete, zero-ambiguity specification for building a Progressive Web App (PWA). Read this entire document before writing any code. Every feature, rule, and edge case is documented. Do not assume anything not stated here.

---

## 1. PRODUCT OVERVIEW

### What is this?
A **mobile-first Progressive Web App (PWA)** for managing the weekly class timetable of a tuition center in Sri Lanka. It serves two user types: **Admins** (who configure everything and build/manage the timetable) and **Teachers** (who view their assigned schedules and receive notifications about changes).

### Core Philosophy
- **Everything is configurable** — grades, subjects, teachers, rooms, time slots, operating days, visibility permissions — nothing is hardcoded.
- **Mobile-first** — designed for phones. Not desktop. Every UI decision prioritizes mobile usability.
- **Settings-driven** — Admins control the entire system through a robust Settings area. Teachers have a clean, simple read-only experience.
- **Real-time** — changes made by admins are reflected instantly for teachers (Convex provides this).

## 3. USER ROLES & AUTHENTICATION

### 3.1 Authentication Flow (Clerk)

1. **Anyone** can sign up with a **username and password** via Clerk.
2. After signup, the account is in a **"pending approval"** state.
3. An **Admin** sees pending accounts in the **Settings → User Management** tab and can **approve or reject** them.
4. Only after admin approval can the user access the app.
5. When approving, the admin assigns the user a **role**: either `admin` or `teacher`.
6. If the user is assigned as a `teacher`, the admin must also **link them to an existing teacher profile** (created in Settings → Teachers).

### 3.2 Role: Admin
- Full access to everything.
- Can see and use the **Settings tab** (hidden from teachers).
- Can create/edit/delete the timetable.
- Can manage all configuration (grades, subjects, teachers, rooms, time slots, operating days).
- Can approve/reject new signups.
- Can assign roles and permissions.
- Can cancel classes, assign substitutes, and trigger notifications.
- Can view any teacher's schedule and the full master timetable.
- Multiple admins are supported (configurable).

### 3.3 Role: Teacher
- **Cannot** see or access the Settings tab.
- Default view: **only their own schedule**.
- **Configurable per teacher:** Admin can grant a specific teacher permission to view the **full timetable (read-only)**. This is a toggle in Settings → Teachers → [Teacher] → "Allow full timetable view".
- Receives in-app and push notifications for schedule changes.
- Cannot edit anything.

## 4. DATA MODEL (Convex Tables)

### 4.1 `grades`
Represents class levels offered at the center.

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated by Convex |
| `name` | string | Display name, e.g., "Grade 6", "Grade 7", "Primary 1", "Primary 2" |
| `sortOrder` | number | For display ordering |
| `isActive` | boolean | Soft delete / disable |

**Notes:**
- Admin creates and names all grades freely (not hardcoded to 6-11).
- Supports primary grades (e.g., "Primary 1" through "Primary 5") and secondary grades (e.g., "Grade 6" through "Grade 11") — but naming is entirely up to the admin.
- Grade groups / categories are NOT needed in v1. Just a flat list of grades with sort order.

### 4.2 `subjects`
Represents subjects taught at the center.

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `name` | string | e.g., "Mathematics", "Art", "Literature" |
| `color` | string | Hex color code for visual identification on timetable |
| `isActive` | boolean | Soft delete / disable |

### 4.3 `teachers`
Represents teacher profiles (separate from Clerk user accounts).

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `name` | string | Full name |
| `phone` | string (optional) | Contact number |
| `subjects` | array of subject IDs | Subjects this teacher can teach (many-to-many) |
| `canViewFullTimetable` | boolean | If true, this teacher can see the full master timetable (read-only). Default: false. |
| `isActive` | boolean | Soft delete / disable |
| `clerkUserId` | string (optional) | Linked Clerk user ID (set when admin approves a signup and links it to this teacher) |

**Notes:**
- One teacher can teach **multiple subjects**.
- One subject can be taught by **multiple teachers**.
- A teacher profile can exist before a Clerk user account is linked to it (admin pre-creates teacher profiles, then links when the person signs up).

### 4.4 `rooms`
Represents physical rooms/halls in the center.

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `name` | string | e.g., "Room A", "Main Hall", "Lab" |
| `capacity` | number (optional) | Max students (for reference, not enforced) |
| `isActive` | boolean | Soft delete / disable |

### 4.5 `timeSlots`
Defines the time grid for scheduling. Configured per day of the week.

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `dayOfWeek` | number | 0 = Sunday, 1 = Monday, ..., 6 = Saturday |
| `slotIndex` | number | Order within the day (1st slot, 2nd slot, etc.) |
| `startTime` | string | "08:00" (24hr format) |
| `endTime` | string | "09:00" (24hr format) |
| `label` | string (optional) | e.g., "Period 1", "Morning Session" |

**Notes:**
- Slots are defined **per day** — Monday can have different slot times than Saturday.
- Duration can vary per slot (e.g., slot 1 = 1 hour, slot 2 = 45 minutes).
- Once configured, these repeat every week automatically.

### 4.6 `operatingDays`
Controls which days the center is open.

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `dayOfWeek` | number | 0-6 |
| `isActive` | boolean | Whether the center operates on this day |

**Notes:**
- All 7 days available, each can be toggled on/off by admin.
- Timetable UI only shows active days.

### 4.7 `timetableEntries`
The core timetable — each row is one class session in the recurring weekly schedule.

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `dayOfWeek` | number | 0-6 |
| `timeSlotId` | ID (ref → timeSlots) | Which time slot |
| `gradeId` | ID (ref → grades) | Which grade |
| `subjectId` | ID (ref → subjects) | Which subject |
| `teacherId` | ID (ref → teachers) | Assigned teacher |
| `roomId` | ID (ref → rooms) | Assigned room |
| `notes` | string (optional) | Any special notes |

**IMPORTANT — No Hard Conflict Blocking:**
- A grade CAN have 2+ classes in the same time slot (because of bucket/elective subjects — e.g., some students take Art while others take Literature at the same time).
- A room CAN have 2+ classes in the same time slot (because classes may be combined).
- A teacher CAN be assigned to 2+ rooms in the same time slot (because some teachers manage parallel classes).
- Instead of blocking, the system should show **soft warnings** when creating entries: "⚠️ Teacher X is already assigned to Room Y at this time" — but still **allow** the admin to proceed.

### 4.8 `scheduleOverrides`
Handles day-specific exceptions to the recurring timetable (cancellations, substitutions).

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `timetableEntryId` | ID (ref → timetableEntries) | Which recurring entry this overrides |
| `date` | string | Specific date "YYYY-MM-DD" |
| `type` | string | `"cancelled"` or `"substitution"` |
| `substituteTeacherId` | ID (ref → teachers, optional) | If substitution, who's replacing |
| `reason` | string (optional) | e.g., "Teacher sick", "Public holiday" |
| `createdBy` | ID | Admin who made this change |
| `createdAt` | number | Timestamp |

### 4.9 `notifications`
In-app notifications for teachers.

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `recipientTeacherId` | ID (ref → teachers) | Who receives this |
| `title` | string | Short heading |
| `message` | string | Details |
| `type` | string | `"cancellation"`, `"substitution"`, `"schedule_change"`, `"general"` |
| `isRead` | boolean | Has the teacher seen it |
| `relatedDate` | string (optional) | The date this relates to |
| `createdAt` | number | Timestamp |

### 4.10 `users`
Links Clerk auth to internal roles/profiles.

| Field | Type | Description |
|---|---|---|
| `_id` | ID | Auto-generated |
| `clerkUserId` | string | Clerk's user ID |
| `role` | string | `"admin"` or `"teacher"` |
| `teacherId` | ID (optional, ref → teachers) | Linked teacher profile (only for teacher role) |
| `isApproved` | boolean | Admin has approved this account |
| `createdAt` | number | Timestamp |

### 4.11 `centerSettings`
Global settings (singleton document or key-value).

| Field | Type | Description |
|---|---|---|
| `centerName` | string | Display name of the center. Default: "Tuition Center" |
| `defaultSlotDuration` | number | Default minutes for new slots, e.g., 60 |

**For Teachers (3 tabs):**
1. **My Schedule** — their timetable (default landing page)

**For Admins (4 tabs):**

The Settings tab is the command center. It uses a list-based navigation pattern (tap to enter each section):

1. **Center Settings** — Center name, default slot duration
2. **Operating Days** — Toggle which days are active (Sun–Sat toggles)
3. **Time Slots** — Configure time slots per day (add/edit/delete/reorder)
4. **Grades** — CRUD grades, reorder
5. **Subjects** — CRUD subjects, assign colors
6. **Teachers** — CRUD teacher profiles, assign subjects, toggle "canViewFullTimetable"
7. **Rooms** — CRUD rooms
8. **Timetable Builder** — The core form for creating/editing timetable entries (see Section 6)
9. **User Management** — Approve/reject signups, assign roles, link teacher profiles

## 6. TIMETABLE BUILDER (Admin — inside Settings)

This is the most critical screen. It must be **form-based and optimized for mobile**.

### 6.1 Builder Flow

**Step 1: Select Day**
- Show only active operating days as selectable chips/buttons (e.g., Mon, Tue, Wed...).
- Selecting a day shows all time slots for that day with their current assignments.

**Step 2: View Day's Slots**
- Display a vertical list of all time slots for the selected day.
- Each slot shows: time range, and a list of classes assigned to it (if any).
- Each class entry shows: grade, subject (with color dot), teacher, room.
- Empty slots show "+ Add Class" button.
- Slots with existing classes show "+ Add Another Class" button (since multiple classes per slot are allowed).

**Step 3: Add/Edit Class Entry**
- Opens a form/modal with these fields:
  - **Grade** — dropdown of active grades
  - **Subject** — dropdown of active subjects
  - **Teacher** — dropdown of active teachers (filtered: show all, but highlight teachers who teach the selected subject)
  - **Room** — dropdown of active rooms
  - **Notes** — optional text field
- **Soft conflict warnings** appear in real-time as admin selects:
  - "⚠️ Mrs. Silva is already assigned to Room B at this time"
  - "⚠️ Room A already has Grade 8 Math at this time"
  - These are warnings only. The admin can still save.
- Save / Cancel buttons.

**Step 4: Edit/Delete Existing**
- Tap on any existing class entry to edit or delete it.
- Swipe-to-delete or a delete button inside the edit form.

### 6.2 Quick View
- At the top of the Timetable Builder, provide a toggle: **"View by Day"** (default) | **"View by Teacher"** | **"View by Room"** | **"View by Grade"**
- "View by Teacher" shows: select a teacher → see all their classes across the week.
- "View by Room" shows: select a room → see all classes in that room across the week.
- "View by Grade" shows: select a grade → see all classes for that grade across the week.
- These are read-only views for admin reference while building.

## 7. TEACHER'S SCHEDULE VIEW

### 7.1 Default: "My Schedule"
- Landing page for teachers after login.
- Shows **today's schedule** by default.
- Display: vertical list of today's time slots with the teacher's assigned classes.
- Each card shows: time range, subject (with color), grade, room, and any notes.
- **Day switcher** at the top: horizontal scrollable day chips (Mon, Tue, Wed...) — only active operating days shown. Tap to switch day.
- **Current time indicator**: highlight the current/next class.
- If a class is **cancelled** for today (via scheduleOverrides), show it with a strikethrough or "CANCELLED" badge and the reason.

- If the teacher IS the substitute, show the class with a "COVERING" badge.

### 7.2 Full Timetable View (if permitted)
- If `canViewFullTimetable` is true for this teacher, show an additional view toggle or tab: "All Classes".
- This shows the master timetable in a read-only grid or list (organized by day and time slot).
- Teacher can filter by: grade, subject, teacher, room.

## 8. SCHEDULE OVERRIDES (Cancellation & Substitution)

### 8.1 Admin: Override Management
Accessible from the **Overrides tab** (admin only).

**Cancel a Class:**
1. Admin selects a **date** (date picker, defaults to today).
2. System shows all scheduled classes for that date (based on recurring timetable + day of week).
3. Admin taps a class → "Cancel this class".
4. Admin enters a reason (optional).
5. System creates a `scheduleOverride` with type `"cancelled"`.
6. System auto-generates a notification for the assigned teacher.
7. Push notification is sent.

**Assign a Substitute:**
1. Admin selects a date.
2. Admin taps a class → "Assign Substitute".
3. Admin selects a substitute teacher from dropdown.
4. Admin enters a reason (optional).
5. System creates a `scheduleOverride` with type `"substitution"`.
6. System auto-generates notifications for:
   - The **original teacher** ("Your Grade 8 Math class on March 24 has been assigned to Mr. Kumar").
   - The **substitute teacher** ("You have been assigned to cover Grade 8 Math on March 24 in Room A").
7. Push notifications sent to both.

### 8.2 Override Display
- Overrides appear on the teacher's schedule view for the affected date.
- Overrides appear with a visual badge (cancelled = red, substitution = orange).
- Past overrides are kept for history but can be hidden from the main view.

- **Notifications tab** shows a chronological list.

### 9.3 Admin: Manual Notification

- Admin can send a custom notification to: a specific teacher, all teachers, or teachers of a specific grade.

## 11. BUSINESS RULES — CRITICAL SUMMARY

1. **No hard conflict blocking.** Teacher in 2 rooms, grade with 2 classes, room with 2 classes — all allowed. Show **soft warnings only**.
2. **Weekly recurring schedule.** Timetable repeats every week. Changes to timetable entries affect all future weeks.
3. **Overrides are date-specific.** Cancellations and substitutions apply to one specific date only, not recurring.
4. **Teacher visibility is per-teacher.** Default: own schedule only. Admin can enable full timetable view per teacher.
5. **Clerk auth with admin approval gate.** No one enters the app without admin approval.
6. **Settings tab is admin-only.** Teachers never see it.
7. **Everything is configurable.** Grades, subjects, teachers, rooms, slots, days — admin can add/edit/delete freely.
8. **Time slots are per-day.** Monday's slots can differ from Saturday's slots in count, times, and duration.
9. **Soft delete pattern.** Use `isActive: false` instead of hard deletes, so historical data is preserved.
10. **Real-time sync.** Convex ensures that when an admin makes a change, teachers see it immediately.

## 13. CONVEX SCHEMA SETUP

Define all tables in `convex/schema.ts` using Convex's schema builder. Key relationships:
- `timetableEntries` references `timeSlots`, `grades`, `subjects`, `teachers`, `rooms` by their Convex IDs.
- `scheduleOverrides` references `timetableEntries` and optionally `teachers` (for substitute).
- `notifications` references `teachers`.
- `users` references `teachers` (optional) and stores Clerk user ID.

Use Convex indexes for common queries:
- `timetableEntries` by `dayOfWeek`
- `timetableEntries` by `teacherId`
- `timetableEntries` by `roomId`
- `timetableEntries` by `gradeId`
- `scheduleOverrides` by `date`
- `scheduleOverrides` by `timetableEntryId`
- `notifications` by `recipientTeacherId` and `isRead`
- `users` by `clerkUserId`
- `users` by `isApproved`

## 14. API FUNCTIONS (Convex)

### Queries (read)
- `getOperatingDays` — returns all operating days with active status
- `getTimeSlotsByDay(dayOfWeek)` — returns slots for a specific day, ordered
- `getAllGrades` — active grades, ordered
- `getAllSubjects` — active subjects
- `getAllTeachers` — active teachers with linked subjects
- `getAllRooms` — active rooms
- `getTimetableByDay(dayOfWeek)` — all entries for a day, populated with grade/subject/teacher/room names
- `getTimetableByTeacher(teacherId)` — all entries for a teacher across the week
- `getTimetableByRoom(roomId)` — all entries for a room across the week
- `getTimetableByGrade(gradeId)` — all entries for a grade across the week
- `getOverridesByDate(date)` — all overrides for a specific date
- `getTeacherScheduleForDate(teacherId, date)` — recurring entries for the day-of-week, merged with any overrides for that date
- `getNotificationsForTeacher(teacherId)` — ordered by createdAt desc
- `getUnreadNotificationCount(teacherId)` — for badge
- `getPendingUsers` — users with isApproved = false
- `getCenterSettings` — global settings

### Mutations (write)
- CRUD for: grades, subjects, teachers, rooms, timeSlots, operatingDays
- `createTimetableEntry(data)` — with soft conflict check (returns warnings)
- `updateTimetableEntry(id, data)` — with soft conflict check
- `deleteTimetableEntry(id)` — hard delete (since overrides reference it; or soft delete with cleanup)
- `cancelClass(timetableEntryId, date, reason)` — creates override + notification
- `assignSubstitute(timetableEntryId, date, substituteTeacherId, reason)` — creates override + notifications
- `approveUser(userId, role, teacherId?)` — approve signup
- `rejectUser(userId)` — reject signup
- `markNotificationRead(notificationId)`
- `sendAnnouncement(title, message, recipientFilter)` — admin sends custom notification
- `updateCenterSettings(data)`

## 15. EDGE CASES TO HANDLE

1. **Teacher deleted/deactivated** who has timetable entries → Show warning to admin, don't auto-delete entries. Admin must reassign manually.
2. **Time slot deleted** that has timetable entries → Block deletion. Show "This slot has X classes assigned. Remove them first."
3. **Grade/Subject/Room deleted** with entries → Same as above. Block if in use.
4. **Substitute teacher also has a class at the same time** → Show soft warning but allow.
5. **Multiple overrides for the same entry on the same date** → Prevent. One override per entry per date (upsert pattern).
6. **Teacher has no classes for a day** → Show friendly empty state: "No classes today! 🎉"
7. **Admin changes recurring timetable** → This affects ALL future weeks. Past overrides remain valid for their specific dates.
8. **User signs up but no teacher profile exists yet** → Admin can either create a teacher profile first and then link, or reject and ask the person to wait.
